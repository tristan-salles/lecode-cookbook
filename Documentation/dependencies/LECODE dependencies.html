This note presents how to install the required dependencies to run LECODE. Most of the guide refers to installation on Mac OSX however the process is quite similar for installation on Linux system.

GENERAL DEPENDENCIES                                   

GNU compilers 
http://hpc.sourceforge.net/

$> unzip gcc-xxx.tar.gz
 
$> sudo mv gcc-xxx.tar /usr/local
$> cd /usr/local
$> sudo tar -xvf gcc-xxx.tar
$> cd /usr/local/bin
$
>
   sudo ln -s /usr/local/gcc-xxx/bin/gcc gcc
$
>
   sudo ln -s /usr/local/gcc-xxx/bin/gfortran gfortran
$
>
   sudo ln -s /usr/local/gcc-xxx/bin/cpp cpp
$
>
   sudo ln -s /usr/local/gcc-xxx/bin/g++ g++
 
in your ~/.profile add 
export PATH=/usr/local/bin:$PATH
export LD_LIBRARY_PATH=/usr/local/lib:$LD_LIBRARY_PATH
Then in terminal:
$
>
   source ~/.profile
 
Then test in the terminal that you are calling the version you've just installed of gcc and gfortran
$
>
   cd ~
$> gfortran --version   it should be the same as    $> /usr/local/gcc-xxx/bin/gfortran --version
$> gcc --version   it should be the same as    $> /usr/local/gcc-xxx/bin/gcc --version

Note: 
For mountain lion I had to copy the lib/ file from gcc-4.8 in the /usr/local/lib folder to get mpich 3.0.2 configured correctly


MPICH3 
http://www.mcs.anl.gov/research/projects/mpich3/

$> tar -xvf mpich3-xxx.tar
$> cd mpich3-xxx
$
>
  CC=gcc FC=gfortran CXX=g++ ./configure --prefix=/usr/local/lecode/mpich --enable-fast=all --enable-shared --enable-sharedlibs=osx-gcc
$
>
   make
$
>
   sudo make install
 
in your ~/.profile add 
export MPI_DIR=/usr/local/lecode/mpich
export PATH=$MPI_DIR/bin:$PATH
export LD_LIBRARY_PATH=$MPI_DIR/lib:$LD_LIBRARY_PATH
export DYLD_LIBRARY_PATH=$MPI_DIR/lib:$DYLD_LIBRARY_PATH

Then in terminal:
$
>
   source ~/.profile

KDTREE 
https://github.com/jmhodges/kdtree2

Create a folder kdtree2 on your desktop and download kdtree.f90 and kdtree2.f90 files in the src-f90 folder from Github.
Then copy the lines below in a Makefile within the kdtree2 folder.
# KD_TREE2
# For gfortran
#
FLAGSOPT= -O3 -funroll-loops --param max-unroll-times=2 -cpp -ftree-vectorize -ffast-math -lstdc++
F90=mpif90
#
FLAGS= ${FLAGSMACHINE} ${FLAGSALWAYS} ${FLAGSOPT}  
MY_DIR=`basename ${PWD}`
all:     kdtree2_test kdtree_test 
     ar -rcs libkdtree.a kdtree.o  kdtree2.o
kdtree2_test:     kdtree2.o
     ${F90} ${FLAGS} -c kdtree2.o
kdtree_test:     kdtree.o
     ${F90} ${FLAGS} -c kdtree.o
%.o :: %.f90
     ${F90} ${FLAGS} -c $<
tar:
     cd ..; tar zcvf ${MY_DIR}.tgz ${MY_DIR}
clean:
     /bin/rm -f *.o *.mod *.il  

Save and close the file and open the kdtree2.f90 file and change lines 22 and 23 to:
!integer, parameter :: kdkind = sp


integer, parameter :: kdkind = dp
to enable double precision.

In a terminal go to the folder location and compile the code
$
>
   cd ~/Desktop/kdtree2
$
>
   make 
 
Then you will need to move the folder to a place that will be pointed out within LECODE configuration file. The default is:
$> sudo mv ~/Desktop/kdtree2 /usr/local/lecode/kdtree2

TRIANGLE
http://www.cs.cmu.edu/~quake/triangle.html
 
Using root privileges sudo move the triangle folder to /usr/local/.
in terminal:
$
>
   sudo mv triangle.tar /usr/local/
$
>
   tar -xvf triangle.tar
 
There are 3 things to do for making Triangle working in Tellus:
  • First you will need to compile it with the Makefile do not use the 'make trilibrary' command but simply 'make'. It will produce the ./triangle & ./showme binaries that could be useful. Do not use the -DLINUX switch for mac compilation.
  • Then go within the triangle.c file and change the main(int argc, char **argv) & main(argc, argv) at lines 15682 & 15684 respectively to many(int argc, char **argv) & mainy(argc, argv). Just to prevent from having multiple main functions when linking to Tellus.
  • Finally compile the changed file using: cc -O -DLINUX -I/usr/X11R6/include -L/usr/X11R6/lib -c -o ./triangle.o ./triangle.c or  cc ls
in your ~/.profile add 
export PATH=/usr/local/lecode/triangle:$PATH
Then in terminal:
$
>
  source ~/.profile

Finally do one of the following depending of your compiler (GNU or Intel): 
$
>
  gcc -O3 -m64 -fno-common -fPIC -c -o triangle.o triangle.c 
$
>
  icc -O3 -m64 -fno-common -fPIC -c -o triangle.o triangle.c 

 
Installing LECODE


 
$> cd ~/workspace
$> git clone
 https://tristan-salles@bitbucket.org/tristan-salles/sgfm.git
$
>
  cd sgfm/LECODE
$
>
  emacs config/Makefile.inc

In the Makefile.inc you will need to specify the CFGFILE (first line) depending on your system.
Several examples of files (Makefile.cfg.osx & Makefile.cfg.linux) are provided and should be changed according to the previous dependencies installation.
 
Once you have set the configuration parameters, run:

$
>
   make clobber
$
>
   make dist

It will create lecode distribution and a binary file located in the bin directory.
 
Note:
if you want to call lecode directly from a terminal you can add in your ~/.profile 
export PATH=/lecode-install-dir/bin:$PATH
export DYLD_LIBRARY_PATH="/lecode-install-dir/build/lib:$DYLD_LIBRARY_PATH" (mac OsX)
export LD_LIBRARY_PATH=/lecode-install-dir/build/lib:$LD_LIBRARY_PATH  (Linux)
Then in terminal:

$
>
   source ~/.profile
